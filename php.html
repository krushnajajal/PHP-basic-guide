Object-Oriented Programming (OOP) Concepts

OOP is a programming paradigm based on the concept of "objects", which can contain data in the form of fields (often known as attributes or properties), and code, in the form of procedures (often known as methods). Here are the fundamental concepts in PHP.

1. Classes and Objects

A class is a blueprint for creating objects. It defines a set of properties and methods that are common to all objects of one type. An object is an instance of a class, with its own specific values for the class's properties.

<?php
class Car {
    public $color;
    public $model;

    public function getDescription() {
        return "This is a " . $this->color . " " . $this->model . ".";
    }
}

$myCar = new Car();
$myCar->color = "red";
$myCar->model = "Volvo";

echo $myCar->getDescription(); // Output: This is a red Volvo.
?>

2. Constructor and Destructor

A constructor (`__construct`) is a special method called when an object is created, used to initialize its properties. A destructor (`__destruct`) is called when an object is about to be destroyed, useful for cleanup tasks like closing database connections.

<?php
class Vehicle {
    public $brand;

    public function __construct($brand) {
        $this->brand = $brand;
        echo "A new {$this->brand} vehicle is created.<br>";
    }

    public function __destruct() {
        echo "The {$this->brand} vehicle is destroyed.<br>";
    }
}

$truck = new Vehicle("Ford");
// The script ends here, so the destructor is called.
?>

3. Inheritance

Inheritance allows a class (child class) to inherit public and protected properties and methods from another class (parent class). This promotes code reusability. A child class can also override a parent's method to provide its own implementation.

<?php
class Animal {
    public function eat() {
        echo "This animal eats food.<br>";
    }
}

class Dog extends Animal {
    // Overriding the parent's eat() method
    public function eat() {
        echo "The dog eats dog food.<br>";
    }
}

$myDog = new Dog();
$myDog->eat(); // Calls the overridden method in the Dog class
?>

4. Serialization

Serialization is the process of converting an object into a string representation, so it can be stored or transmitted. `serialize()` converts the object to a string, and `unserialize()` converts it back. The magic methods `__sleep()` and `__wakeup()` can be used to manage resources during this process.

<?php
class User {
    public $name;
    private $db_connection;

    public function __construct($name) {
        $this->name = $name;
    }

    public function __sleep() {
        // Return array of properties to serialize
        return ['name'];
    }

    public function __wakeup() {
        // Re-establish resources, e.g., DB connection
    }
}

$user = new User("John Doe");
$serializedUser = serialize($user);
echo $serializedUser;
?>

5. Magic Methods

Magic Methods are special methods prefixed with `__` that are executed in response to certain events. For example, `__toString()` is called when an object is treated as a string.

<?php
class Book {
    public $title;
    public $author;

    public function __construct($title, $author) {
        $this->title = $title;
        $this->author = $author;
    }

    public function __toString() {
        return "'" . $this->title . "' by " . $this->author;
    }
}

$book = new Book("1984", "George Orwell");
echo $book; // Automatically calls __toString()
?>

MySQL with PHP: MySQLi vs. PDO

PHP offers two primary ways to connect to a MySQL database: the MySQLi (Improved) extension and PDO (PHP Data Objects). PDO is generally preferred for its flexibility, as it can connect to various database systems, not just MySQL.

MySQLi (Object-Oriented)

<?php
$servername = "localhost";
$username = "user";
$password = "pass";
$dbname = "myDB";

$conn = new mysqli($servername, $username, $password, $dbname);

if ($conn->connect_error) {
  die("Connection failed: " . $conn->connect_error);
}

$stmt = $conn->prepare("SELECT id, name FROM users WHERE id = ?");
$stmt->bind_param("i", $id);
$id = 1;
$stmt->execute();
$result = $stmt->get_result();
$user = $result->fetch_assoc();

$stmt->close();
$conn->close();
?>

PDO

<?php
$servername = "localhost";
$username = "user";
$password = "pass";
$dbname = "myDB";

try {
  $conn = new PDO("mysql:host=$servername;dbname=$dbname", $username, $password);
  $conn->setAttribute(PDO::ATTR_ERRMODE, PDO::ERRMODE_EXCEPTION);

  $stmt = $conn->prepare("SELECT id, name FROM users WHERE id = :id");
  $stmt->execute(['id' => 1]);
  $user = $stmt->fetch(PDO::FETCH_ASSOC);

} catch(PDOException $e) {
  echo "Connection failed: " . $e->getMessage();
}
$conn = null;
?>

XMLHttpRequest (XHR) Object

The XHR object is the core of AJAX. It is used to send HTTP requests to a server and receive responses.

Key Properties:

  • onreadystatechange: Function called when `readyState` changes.
  • readyState: Holds the request status (0-4).
  • status: HTTP status code (e.g., 200, 404).
  • responseText: Response as a string.

Key Methods:

  • open(method, url, async): Initializes a request.
  • send(body): Sends the request.
  • setRequestHeader(h, v): Sets an HTTP header.
function loadData() {
  const xhttp = new XMLHttpRequest();
  xhttp.onreadystatechange = function() {
    if (this.readyState == 4 && this.status == 200) {
      document.getElementById("demo").innerHTML = this.responseText;
    }
  };
  xhttp.open("GET", "data.txt", true);
  xhttp.send();
}
